Game to Completion


Upon starting the game, a menu will appear, prompting the user to select one of the six available modes. Only inputs within the range of 1 to 6 will be accepted, and any invalid input will be rejected and prompt the user to try again.

Detailed Description of how artefact works and how it will meet my objectives

1 Single player Mode:
  • In this mode, the user plays against the dealer (i.e., computer). The objective is to get a higher total than the dealer without exceeding 21. When selected, the code asks for your bet, from the 1000 in your balance. This allows the user to earn or lose money.
  • Then the dealer gives the user two cards. The user can either hit (ask for another card) or stand (keep the current hand) to try and get closer to 21. The dealer will continue to hit until their total is 17 or higher. If the user's total exceeds 21, they lose, and if the dealer's total exceeds 21, the user wins. User will input “h” or “s”.     Modelling

    I implemented game modelling in the Python blackjack game by designing and programming the game interface, creating algorithms for single player, multiplayer, and simulation modes, and incorporating statistical analysis to enhance the user's gaming experience. I also used a progress bar for aesthetics and to give the user some time to process outputs.


    Abstraction

    The single-player mode against the computer and no betting feature provided the foundational structure for the project. This implementation allowed for the game's mechanics to be developed, tested, and refined before adding more advanced features, such as multiplayer or betting.
